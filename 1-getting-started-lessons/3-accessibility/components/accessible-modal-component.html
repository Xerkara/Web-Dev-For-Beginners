🎯 Accessible Modal Dialog Component

This component demonstrates proper accessibility implementation for modal dialogs following WCAG 2.1 AA standards. Features include focus trapping, keyboard navigation, ARIA attributes, and comprehensive error handling.

✨ Accessibility Features

  • Focus Trapping: Keyboard focus stays within the modal when open
  • ESC Key Support: Press ESC to close the modal
  • Click Outside to Close: Click the overlay to dismiss the modal
  • ARIA Attributes: Proper roles, aria-labelledby, aria-describedby
  • Focus Indicators: 3px outline with 2px offset (WCAG AA standard)
  • Keyboard Navigation: Tab order managed within modal
  • Form Validation: Real-time and submission validation
  • Error Handling: Color + icon (compliant with color-only WCAG rule)
  • Touch Targets: Minimum 44x44px interactive elements
  • Screen Reader Support: Semantic HTML and ARIA live regions

Dialog Title

This is a simple modal dialog demonstrating accessibility best practices.

Try these interactions:

  • Press Tab to navigate through buttons
  • Use Enter/Space to activate buttons
  • Press Escape to close
  • Click outside to close
